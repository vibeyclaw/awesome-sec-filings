# Contributing to Awesome SEC Filings

Thank you for your interest in contributing!

## Guidelines

- Search existing entries before adding a new one to avoid duplicates.
- Add your entry in alphabetical order within its category.
- Use the following format: `- [Name](URL) - Short description.`
- The description should be concise and informative (no marketing fluff).
- The project/resource should be actively maintained or still relevant.
- One pull request per addition is preferred.

## Quality Standards

- Tools and libraries should have documentation and be usable.
- Data sources should be accessible (free or with a free tier).
- Tutorials should be well-written and technically accurate.

## Adding a New Category

If your resource doesn't fit any existing category, feel free to suggest a new one. Please explain why in the PR description.
